# OpenWork

一个从公开招聘帖里发现全球工作机会的 3D 地球网站。

## 连续场景

- `index.html`：首页与搜索地图共用同一个场景、同一颗地球。
- 点击“开始查看”后，地球从右侧平滑移到主视野，首页文案退场，搜索栏从底部出现；过程中不跳页、不重建地球。
- 搜索状态采用“上方岗位浮卡、下方局部地球、底部单行搜索条”的构图；浮卡可左右切换岗位，并带动地球转向对应地点。
- `explore.html`：仅用于兼容旧链接，会把原有查询条件带回连续场景。

## 数据

当前原型收录 1,495 个最近 30 天的公开招聘线索，其中 902 个为明确远程岗位。来源包含 X、Reddit、领英、AnySearch 全球职位搜索、Remote OK、Remotive、Arbeitnow、We Work Remotely、Himalayas、Jobicy、NoDesk，以及 Apple、Microsoft、NVIDIA 等公司官方招聘页。每条数据都保留原帖或职位页链接。

其中 769 个岗位能落到 259 座具体城市。中国大陆现有 31 个经过官方招聘页核实的岗位，覆盖北京、上海、深圳、苏州与成都。地图只接受两类坐标：职位正文明确写出的城市，或招聘来源中能核实的公司总部城市。只写国家、地区、时区或“全球远程”的岗位仍可搜索和浏览，但不会再用国家中心或海上坐标冒充城市点。

远程岗位不限定方向，除设计、开发、人工智能外，还覆盖客服、行政、财务、教育、医疗、写作、法务、销售、市场和日常运营。

- 圆钉颜色：岗位类型；钉头使用纯色平面材质，短针贴近地球表面
- 圆钉大小：随视角缩放并在远近两端限制尺寸，方便鼠标和触屏点击
- 圆钉聚合：仅在多个岗位的地图位置确实接近时出现；相对默认视角放大达到 40% 后自动拆分
- 圆钉位置：职位城市；若使用公司总部，悬停信息会明确标注
- 国家交互：点击国家后，上方岗位卡会切换为该国岗位合集

底部只保留关键词搜索框；输入“远程”、职位、公司或城市即可缩小地图上的岗位范围。

城市坐标可重新生成：

```bash
node scripts/enrich_locations.mjs
```

## 本地运行

```bash
npm install
python3 -m http.server 4182
```

打开 `http://127.0.0.1:4182`。

## 小红书小工具版本

小红书版本沿用同一套界面和岗位数据，但改为完全离线的单入口页面：地球、国家边界、城市点与岗位信息都随安装包加载，不发起网络请求，也不打开站外链接。它同样可直接作为移动端网页运行。

生成待上传的压缩包：

```bash
npm run build:xhs
npm run validate:xhs
```

产物位于 `releases/openwork-xhs.zip`，提交表单可使用 `releases/openwork-icon.png` 作为小工具图标。国家边界已固化在项目内，打包不依赖网络。每周重新收集并核对岗位数据后，再依次运行构建与校验命令，就会生成包含最新数据的新版本；网页版本不会因此受到影响。
